
		Курс - DevOps Tools для разработчиков
	
	CI - автоматизация интеграции изменений кода от нескольких контрибьюторов в единый программный продукт
	
		- управление процессом
		- контроль качества
		- сокращение цикла обратной связи
		
	Когда CI нет:
		
		- ручное управление
		- медленные процессы
		- поспешность в разработке
		- низкое качество
		- снижение доверия
		
	Основные критерии CI
	
		- единый источник стабильной версии ПО, например main branch
		- автомат. проверки кода в различных источниках
		- масштабируемость 
			- по кол-ву сотрудников
			- по кол-ву изменений
			
	Trunk based development
	
		- 1 стаб master
		- очень короткие feature branches
		- регулярные rebase/merges
		
		Простые ревью
		Минимум конфликтов
		Очень короткий цикл обратной связи
		
		- Требует предварительной проработки задач
		- Требует коммуникации внутри команды
		
	DoD - definition of done
	
	Multi remote repositories
	
	
		Jenkins
		
	Master (server) - static slaves (nodes) + dynamic slaves (clouds)
	
	
		Code - commit - pipelines
		
	CI pipeline - build, unit test, integration tests
	
	CD pipeline - review, staging, production

	- разделение ответственности
	- снижение рисков
	- сокрашение цикла обратной связи
	- реализация среды
